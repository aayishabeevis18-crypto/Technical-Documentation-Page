Introduction

This documentation gives a concise overview of fundamental web technologies used to build accessible, responsive user interfaces: HTML, CSS, and JavaScript.

Each section below contains short examples and recommended practices. Use the left navigation to jump to any topic.

HTML Basics

HTML (HyperText Markup Language) defines the structure of web pages. Pages use elements such as <header>, <nav>, <main>, <section>, and <footer>.

Use semantic elements where possible: a <button> for interactive actions, a <form> for inputs, and meaningful headings for document sections.

Example

<section id="example">
  <header>Example Header</header>
  <p>Some content here</p>
</section>
  • Use semantic tags for meaning
  • Provide descriptive alt text for images
  • Use headings in hierarchical order (h1 → h2 → h3...)
CSS Basics

CSS controls presentation: layout, colors, spacing, and responsive behavior. Use rules like display: flex;, grid-template-columns, and media queries to adapt to different screens.

Keep styles modular and prefer classes over styling elements directly when reusability is needed.

Example

/* simple responsive container */
.container {
  display: flex;
  flex-wrap: wrap;
}
  • Use rem or em for scalable typography
  • Prefer mobile-first media queries
JavaScript and DOM

JavaScript adds interactivity. Use the DOM API to query and update elements. For example: document.querySelector('.btn') returns the first element with class btn.

Keep DOM updates minimal — batch changes and avoid expensive reflows. Delegate event handlers when possible.

Small code snippets

// add a click handler
document.querySelector('#saveBtn').addEventListener('click', function () {
  // do something
});
  • Prefer unobtrusive JavaScript (separate from HTML)
  • Use defer on script tags for non-blocking load
Accessibility and Best Practices

Accessibility ensures all users can use your site. Include ARIA attributes when native HTML cannot convey semantics, e.g., role="navigation" or aria-label.

Always provide keyboard focus states and logical tab order. Use color contrast sufficient for readability.

Checklist

  • Provide alt on images or use role="presentation" for decorative images
  • Label form controls explicitly with <label for="...">
  • Test with keyboard only and with screen readers
  • Use semantic HTML before adding ARIA attributes

Example of an accessible button: <button aria-label="Close dialog">✕</button>
